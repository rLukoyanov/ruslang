# Тестовые файлы для компилятора

## Положительные тесты (корректные программы)

### Базовые возможности

- **test_simple.model** - простейшая программа с одной переменной
- **work.model** - базовый пример с несколькими типами данных
- **floats.model** - тест вещественных чисел

### Операторы

- **test_arithmetic.model** - арифметические операции (plus, min, mult, div)
- **test_logic.model** - логические операции (and, or, ~, сравнения)
- **test_if_else.model** - условные операторы if-else
- **test_for_loop.model** - циклы for с шагом
- **test_while_loop.model** - циклы while
- **test_io.model** - операции ввода (readln) и вывода (writeln)

### Типы данных

- **test_number_systems.model** - различные системы счисления (двоичная, восьмеричная, шестнадцатеричная, десятичная)
- **test_float_operations.model** - операции с float, включая научную нотацию

### Структура программы

- **test_compound_statements.model** - составные операторы (begin-end)
- **test_complex_expressions.model** - сложные выражения со скобками
- **test_all_operators.model** - комплексный тест всех операторов
- **test_edge_cases.model** - граничные случаи

## Отрицательные тесты (программы с ошибками)

### Семантические ошибки

- **error_undeclared_variable.model** - использование необъявленной переменной
- **error_redeclaration.model** - повторное объявление переменной
- **error_type_mismatch.model** - несоответствие типов при присваивании

### Синтаксические ошибки

- **error_missing_semicolon.model** - отсутствие точки с запятой
- **test_error.model** - общий тест ошибок

### Лексические ошибки

- **error_invalid_number.model** - неверный формат чисел в разных системах счисления
- **lexica_errors.model** - различные лексические ошибки

## Как использовать

Для запуска теста:

```bash
go run ./ ./examples/<имя_файла>.model
```

Для запуска всех тестов:

```bash
for file in examples/*.model; do
    echo "Testing: $file"
    go run ./ "$file"
    echo "---"
done
```

## Ожидаемые результаты

**Положительные тесты** должны:

- Успешно пройти лексический анализ
- Успешно пройти синтаксический анализ
- Успешно пройти семантический анализ
- Завершиться с кодом 0

**Отрицательные тесты** должны:

- Выявить соответствующие ошибки на нужном этапе
- Показать сообщение об ошибке с номером строки и столбца
- Завершиться с кодом 1
